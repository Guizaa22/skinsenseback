================================================================================
                          DESIGN DECISIONS
================================================================================

✓ Package-by-Feature
  → Modules indépendants pour développement parallèle
  → Pas de conflits sur les packages
  → Chaque module auto-contenu (entity/repo/service/controller/dto)

✓ Constructor Injection Uniquement
  → @RequiredArgsConstructor (Lombok)
  → Pas d'injection par field
  → Dépendances claires au instantiation

✓ DTOs for API Boundaries
  → Controllers ne retournent pas d'entities
  → Validation au niveau DTO
  → Separation of concerns

✓ Transactional Services
  → @Transactional sur les services
  → Atomicity garantie
  → Rollback on unchecked exceptions

✓ UUID Primary Keys
  → Pas de leak d'IDs
  → Meilleure distribution
  → Ready for sharding

✓ Automatic Timestamps
  → @PrePersist/@PreUpdate
  → Audit trail built-in
  → createdAt, updatedAt sur toutes les entities
